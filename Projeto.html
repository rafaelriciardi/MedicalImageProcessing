Objetivo:

Para suprir as necessidades em pesquisas de ressonncia magntica fazendo testes em animais como em ratos de laboratrio, tem se buscado substituir os animais por modelos, como a utilizao de phantoms que aproxime as caractersticas e propriedades do animal real, imitando o comportamento de suas caractersticas na aquisio de imagens.

Assim, o objetivo do projeto conseguir gerar um modelo a partir de imagens da ressonancia magntica de um rato, aplicando tnicas de processamento de imagens, gerao de STL com a construo do mesh e impresso 3D do mesmo.

Filtrando as imagens¶

Nessa etapa, o primeiro grande desafio foi justamente aprender a trabalhar e manipular as propriedades das imagens DICOM, para uma vez que tivessemos aplicado algum filtro, pudessemos salv-la com sucesso. Aps aprender algumas particularidades sobre o formato, criamos a funo save_dicom_file. Que recebe o arquivo DICOM original da imagem, o novo array e o caminho a ser salvo. Alm disso, para que possa ser salvo, temos que alterar tambm a propriedade TransferSyntaxUID, que para uma mais permissiva s alteraes.
Alm disso, aplicamos os seguintes filtros:

  • intensity_filter: Com base na pr-visualizao das imagens, percebemos que filtrar os pixels com intensidade inferiores a 75 ajudava bastante no rudo original. Ento, criamos essa funo que transforma todos os pixels menores ou iguals a 75 em 0.
  • erosion: Remove alguns pixels das camadas nas partes de contorno internas e externas da imagem, aumentando os buracos e espaamentos das regies e removendo pequenos detalhes. Alm de remover os detalhes que prejudicavam a construo do mesh, foi bem til tambm para remoo dos rudos restantes.
  • dilatation: Realiza justamente o processo inverso da eroso, adicionando pixels s partes de contorno internas e externas da imagem. No caso, o tamanho do filtro aplicado na dilatao foi maior, pois percebemos que melhorava a reconstruo da imagem no sentido de faz-la com ainda menos detalhes nos contornos, o que beneficiava o mesh gerado posteriormente.
  • noise_filter: possvel observar qualitativamente que nas regies centrais da imagem a relao sinal rudo (SNR) maior em relao s partes mais distantes (isso ocorre devido homogeneidade do campo ser maior no centro da bobina), e esse rudo precisa ser minimizado para no gerar pontos fora da rea de interesse. Para isso foi utilizado um filtro passa baixas no domnio do espao (por escolha, sendo que o filtro no domnio das frequncias obteria um resultado igual ou muito semelhante) de tamanho 5x5. A escolha da dimenso do filtro foi aps ser comparado com filtros 3x3 e 7x7, tendo melhor resultado do que o 3x3 e semelhante ao 7x7. O filtro passa baixa diminui a resoluo das estruturas internas que so menores, porm como o objetivo realizar a segmentao externa esses detalhes perdidos no prejudicam. O filtro tambm suaviza os contornos do corpo externo, contribuindo para o processo.

def save_dicom_file(original_file, new_array, file_path):

    original_file.file_meta.TransferSyntaxUID = ExplicitVRLittleEndian
    original_file.PixelData = new_array.tobytes()
    original_file.save_as(file_path)

def intensity_filter(pixel_array, value=75):
    pixel_array = np.array(pixel_array)
    pixel_array[pixel_array<=value] = 0
    return pixel_array

def noise_filter(img):
    img = np.array(img)
    w = np.ones([5,5])/25
    Im_filter = ndimage.convolve(img,w, mode='reflect')
    return Im_filter

def erosion(img):
    kernel = np.ones((5,5), np.uint8)
    img_erosion = cv2.erode(img, kernel, iterations=1)
    
    return img_erosion

def dilatation(img):
    kernel = np.ones((10,10), np.uint8)
    img_dilation = cv2.dilate(img, kernel, iterations=1)
    return img_dilation

Teste com imagens binarizadas¶

Para tentar melhorar ainda mais o resultado final do mesh, como o processo de impresso no ve os nveis ou dureza dos tecidos, apenas se h ou no material a ser depositado no local, tentamos fazer uma converso binria na imagem, transformando todos os pixels maiores que 0 em 255.

def binarize_image(pixel_array, threshold=0):
    pixel_array = np.array(pixel_array)
    pixel_array[pixel_array<=threshold] = 0
    pixel_array[pixel_array>threshold] = 255
    return pixel_array

Gerando o mesh e o arquito STL¶

Para o processo de gerao do arquito STL, utilizamos a implementao encontrada no git https://github.com/dave3d/dicom2stl.

Como a implementao utiliza como entrada uma imagem com a extenso nii.gz, fazemos a converso dos dados que tinhamos para esse formato e com o auxilio das bibliotecas sitk e itkwidgets conseguimos ja visualizar a forma do rato em 3D, mas ainda sem gerarmos o mesh e o STL.

Para fins de comparao, utilizamos a funo implementada para gerao do primeiro STL, com os parmetros padres e recomendados na documentao e posteriormente, fomos testando novos parmetros e valores, principalmente no que diz respeito a suavizao da forma, para gerar uma superfcie mais natural no objeto, o que nos leva ao resultado final.
